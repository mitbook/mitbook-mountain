#框架描述

   是把日志数据存放在redis中,当日志数据量比较大的时候数据基于纯内存操作,有助于提升框架的性能
#使用说明
 1.详细请看mitbook-test工程(基于订单业务)
#设计原理
 1.目前不提供原理图,如果了解seata框架的话,本框架思想也是比较简单的,因为该框架是基于seata框架的
#支持数据库
 1.市面上的数据库都支持,只要是符合jdbc规范的数据库都支持
#技术选型
 1.springboot,aop,redis等
